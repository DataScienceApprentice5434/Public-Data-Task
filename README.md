## Public-Data-Task
A task for my apprenticeship, analysing a public data set. I chose to analyse a [Warhammer 40k dataset](https://www.kaggle.com/datasets/theredmage/warhammer-40k).


# Executive Summary
Warhammer 40,000 is an ongoing tabletop game involving various factions and characters, with each faction holding stories and details that have been created over the course of 38 years since its original launch in 1987. Throughout the years however there have been many claims that certain factions are favoured by Games Workshop, receiving more lore (stories and details) and models than other factions. The purpose of this analysis is to determine the authenticity of such claims, breaking down various factors to determine whether one faction receives more content and models than others. This will be done through a Kaggle dataset “Warhammer 40k”. This dataset was used as it extensively documented the various factors of Warhammer 40,000 and the models produced for it. The data portrayed attributes which would be invaluable to determining any trends in model and faction bias within Games Workshop such as pricing of models and the faction each belonged to. The data was determined to be of adequate quality on initial inspection through data checks to ensure key data was present along with the sources of the data being documented within the datasets themselves. However some data was missing and data cleaning would be required to draw accurate results from the data sets. Additionally the data provided was specifically for the newest edition of Warhammer, meaning that historical insights would not be possible to be drawn from the data, but that the data was up to date and could provided current insights, having last been updated on the 10th March 2025.

# Data Processing and Cleansing
Initially, the data was in separate files, requiring them to be combined in order to gather the full data in one centralised sheet. 

![Files list](https://github.com/DataScienceApprentice5434/Public-Data-Task/blob/main/images/All%20Files.png)
 
To draw relationships between the data, a primary key was determined. An identifier is present in every sheet and so will be used as a primary key going forward. The next step was to combine the data into one centralised sheet containing all of the data, to allow for detailed analysis of all appropriate factors. This required the creation of a new spreadsheet holding all of the column names present in each spreadsheet as doing so would allow for all data to be consolidated into one table and formatted in a way that would allow detailed breakdowns and comparisons of each model and faction present within the data. After creating the central spreadsheet labelled “Framework”, PowerBI was used to merge each spreadsheet into one central spreadsheet. PowerBI was used for its data manipulation and querying capabilities along with then allowing data visualisation. PowerBI has limitations that did affect the methods and implementation of the analysis, such as the incapability to edit individual sets of data wherein inconsistencies were discovered, instead requiring the inconsistency to be traced to the dataset it originated from and edited there. However its capabilities outweighed the limitations through allowing comprehensive manipulation of the data through DAX and PowerQuery alongside providing extensive options for visualisation and comparing data to draw conclusions and portray a compelling data story. This was done through using Powerquery to access the folder housing the datasets and using a “Transform File” command to extract the data from the CSVs. Then, using the Framework file as the “Sample File”, Table.ExpandTableColumn was used to draw the data from each CSV and place it in the appropriate column, with the equation being “= Table.ExpandTableColumn(#"Removed Other Columns1", "Transform File", Table.ColumnNames(#"Transform File"(#"Sample File")))”. This left a file with each row from each individual CSV, however this meant there were many duplicate values and the data wasn’t linked, in order to link the data the central file needed to be unpivoted using the id column after removing the example data on the top row. This was done with the query “= Table.UnpivotOtherColumns(#"Removed Top Rows", {"id"}, "Attribute", "Value")” which unpivoted the table to lay it out as such:

![Unpivoted table](https://github.com/DataScienceApprentice5434/Public-Data-Task/blob/main/images/PowerBI%20Unpivoted.png)
 
Through holding each column as an “Attribute” and a “Value”, it allows efficient analysis of the individual aspects of id as each id only holds one entry for each attribute. This is due to how PowerBI reads data, as through consolidating each column into an “attribute” and a “value”, null responses are filtered out along with providing a linear format for the data as opposed to various segments of data being present in different rows. The next step was to do a final cleanse of the data. To begin, rows with null ids were removed, as they provided no insight into trends of Warhammer 40,000 faction bias and would reduce the integrity of the data as there could be duplicates or erroneous data not detected due to a missing id. Additionally, a separate table called “Factions” was added. This table held the full name of each faction along with the faction ID, allowing each entry’s faction id to be replaced by the full name of the faction, in order to remove any confusion or unreadability from the report.

# Data Analysis
After cleaning and compiling the data, analysis could begin. The initial insight to draw was determined to be the number of models owned by each faction. To draw this insight, a graph was made of the count of ids, filtering attribute to solely be “faction id” and have the value be the legend. This created a graph demonstrating the number of models in each faction and immediately portrayed a trend. According to the data, there were 305 Space Marine models, abilities and lore in circulation, over double the number of models the second largest faction had; the Genestealer Cults with 146 models. A pie chart was then generated using the data to draw a more accurate comparison between the number of models each Faction hold and the percentage that is of the total number of models. The pie chart demonstrated that Space Marines were 16.28% of all models created, a large segment when compared to the number of factions present; 23. If all models were evenly split amongst the factions, each faction would account for 4.35% of the total models. Overall, this suggests that the current number of Space Marine models would be 3.74 times more than what an even split would be, demonstrating a clear present bias amongst the models. 

![Model Table](https://github.com/DataScienceApprentice5434/Public-Data-Task/blob/main/images/Models%20per%20Faction.png)
 
This conclusion demonstrates a strong bias towards the Space Marine faction from Games Workshop, however there lies a potential for this data to be inaccurate or skewed from missing context or information. As such further insight is needed to accurately determine whether there is a bias present or if this is from an unknown skew. To determine this, a new visualisation was created to portray the number of “legends” each faction had. Legend in the context of the data was referring to stories or traits that had been given to a model beyond the traditional description of what it was, meaning extra time and effort had been given to portray this model and it held a position within the story telling of Warhammer 40,000. Upon visualising the count of legends per each faction, a similar trend emerged as Space Marines held 276 legends with the second highest count of legends belonging to a faction called “Astra Militarum” with 103 legends, once again less than half the number of legends held by Space Marines. This demonstrates a clear bias within the release patterns of models and lore for Warhammer 40,000. The faction with the least lore and models released was the “Adeptus Titanicus”, with only 4 models and lore entries released.

# Conclusion
Overall, the data suggests a clear bias from Games Workshop towards the Space Marine faction, demonstrating the claims made by the Warhammer community to be at least partially accurate. Additionally the bias was observable in the factions receiving the least models, legends and other material, with Adeptus Titanicus only accounting for 0.21% of the models currently available. The dashboard and analysis taken provides an insight into various aspects of the Warhammer 40,000 table top game, and while being initially taken to determine the presence of bias within the content generation of factions, can be used to determine various factors such as the best faction to begin collecting, with Adeptus Titanicus holding the least amount of models to collect and Tau Empire, with 72 models, being the closest to the average number of models. These insights can be used by collectors and to predict release patterns from Games Workshop, for example with the data analysed it is likely that a new Space Marine model or lore entry could be released as the bias and precedent is demonstrated. Additionally the data holds information such as which factions hold the most unique attributes such as “Damage Descriptors”, which determine playstyle based on how the models are damaged in the game.  Overall the analysis provided concludes that there is bias present within Games Workshop with the creation of their models and lore in the present version of the game. However this analysis is limited through it only being of the current edition, and in the future should include historical data to draw trends over time and predict potential future trends.
